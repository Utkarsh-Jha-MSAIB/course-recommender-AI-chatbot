# Fordham AI Course Recommender Chatbot

An AI-powered course recommendation chatbot for Fordham University students. The system guides students through a short conversational questionnaire, then uses semantic search and Google Gemini to recommend Coursera courses tailored to their background, interests, and schedule.

---

## What It Does

Students chat with the assistant and answer five questions:

1. **Background** — major or academic field
2. **Topic area** — e.g., AI / Machine Learning, Cloud / Deployment, Data Analysis, Business
3. **Focus skill** — selected from a curated dropdown or typed freely
4. **Time available** — how many hours they can commit
5. **Provider preference** and **difficulty level**

Once all answers are collected, the system builds a semantic search query, retrieves matching courses from a FAISS vector index, re-ranks them using a composite scoring formula, and uses Gemini to generate a personalized explanation for each recommendation. Results appear as flashcards in a side panel.

---

## How the Recommendation Works

### 1. Semantic Search
Courses are embedded using `all-MiniLM-L6-v2` (384-dimensional vectors) and stored in a FAISS flat index. At query time, the user profile is converted into a natural language query and encoded with the same model. The top 20 nearest neighbors are retrieved using inner-product similarity.

### 2. Filtering
Candidates are filtered by:
- Maximum learning hours (based on user's time preference)
- Difficulty level (if specified)
- Minimum rating threshold

### 3. Re-ranking
Each candidate receives a composite score:

| Component | Weight | Description |
|-----------|--------|-------------|
| Semantic similarity | 65% | Cosine similarity between query and course embedding |
| Rating | 15% | Normalized course rating (out of 5.0) |
| Popularity | 12% | Log-normalized enrollment count |
| Time fit | 8% | Alignment between course hours and user's preferred duration |

### 4. AI Explanation
The top 3–5 courses are passed to Gemini (`gemini-2.5-flash`) with the user's full profile. Gemini generates a one-sentence rationale and a trade-off note for each course. If Gemini's response is incomplete, a template-based fallback is used.

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Web framework | Flask + flask-cors |
| Semantic search | FAISS (`faiss-cpu`) + SentenceTransformer |
| Embedding model | `all-MiniLM-L6-v2` |
| AI generation | Google Gemini API (`google-generativeai`) |
| Data processing | pandas, numpy, openpyxl |
| Frontend | Vanilla HTML / CSS / JavaScript |

## Setup

### Prerequisites
- Python 3.9+
- A Google Gemini API key ([get one here](https://aistudio.google.com/apikey))

### 1. Create and activate a virtual environment
```bash
python3 -m venv .venv
source .venv/bin/activate        # macOS / Linux
# .venv\Scripts\activate         # Windows
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Configure environment variables
Create a `.env` file in the project root:
```
GOOGLE_API_KEY=your_api_key_here
GEMINI_MODEL=gemini-2.5-flash
```

### 4. (Optional) Rebuild the course index
The `course_artifacts/` directory is already included. Only run this if you update the course data:
```bash
python build_course_catalog.py
```

## Running the App

> **Note for macOS users:** Port 5000 is occupied by AirPlay Receiver. The app runs on port **5001** by default.

```bash
cd course_chatbot
python app.py
```

Open your browser at [http://localhost:5001](http://localhost:5001).

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/` | Serves the chat UI |
| `POST` | `/chat` | Processes a user message and returns a response and recommended courses |
| `POST` | `/clear_history` | Clears the chat history for a given session |

### POST `/chat`

**Request body:**
```json
{
  "message": "AI / Machine Learning",
  "session_id": "abc123"
}
```

**Response:**
```json
{
  "session_id": "abc123",
  "response": "Great choice — AI / Machine Learning can branch...",
  "courses": [
    {
      "course_name": "Advanced Prompt Engineering for Everyone",
      "domain_partner": "Data Science",
      "hours_rating": "8 hours",
      "skills": ["Prompt Engineering", "LLM Application", "Generative AI"],
      "semantic_score": "0.66",
      "final_score": "0.73",
      "url": "https://www.coursera.org/...",
      "image_url": "https://..."
    }
  ]
}
```

Courses are only returned in the final response after all five questions are answered. Earlier turns return an empty `courses` array.

## Course Data

The course catalog is sourced from Coursera and contains approximately **1,600 courses** across domains including:
- AI / Machine Learning
- Data Science & Analytics
- Cloud Computing & Deployment
- Business & Analytics
- Software Development

Each course record includes: name, partner/university, difficulty level, estimated hours, rating, enrollment count, skill tags, description, course URL, and thumbnail image URL.

## Conversation Logging

All conversations are appended to `course_chatbot/logs/chat_history.csv` with the following columns:

| Column | Description |
|--------|-------------|
| `timestamp` | ISO 8601 datetime |
| `session_id` | Unique session identifier |
| `user_message` | Raw user input |
| `assistant_message` | Full bot response including state markers |

## Known Issues

- **Course name not shown on flashcard** — The card UI currently displays domain, duration, skills, and scores but does not render the course title.
- `google.generativeai` package is deprecated — Gemini integration should be migrated to the `google.genai` package in a future update.
- Python 3.9 is past end-of-life for some Google SDK dependencies. Upgrading to Python 3.10+ is recommended.
